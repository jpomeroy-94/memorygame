Program Description
memorygame.htm
  • Events are in a separate javascript file(jsSelect.js)
  • Standard Style Sheets(css) are defined in a separate style sheet file(memorygame.css)
  • Has Jquery interface(http://code.jquery.com/ui/1.9.0/themes/base/jquery-ui.css)
  • Several javascript objects are created within linked javascript files(/jslib/MemoryGame.js, /jslib/FlipCard.js, /jslib/ServerIo.js, /jslib/TrackScore.js)
  • The html5 canvas object is the main tool for running the game. The new html5 feature allows for painting the game only after all images have been loaded from the server to the client giving it a much more professional look and feel.
MemoryGame.js
TrackScore.js
  • Uses concept of closure. Public methods are defined by being located in an object which is returned by the function. Private methods and variables are defined within the function.
FlipCard.js
  • Uses the 'prototype object' for memory sharing. The Constructor(FlipCard) is instantiated/replicated to an array of objects(flipCard). All code which is used by all objects is put into the prototype object of the constructor. All variables, arrays, and objects which are unique to each object are stored outside of the prototype object but in the constructor, thus being placed in every object created.
memorygame.css
  • Style sheet setups are in their own file.
jsSelect.js
  • Event definitions are in their own file.
